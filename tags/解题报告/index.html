第十届河海大学程序设计竞赛高年级组

写在前面

  • 是第一次出题，也可能是最后一次出题。暴露了很多问题，打算好好记录一下这次出题的经验，可能会很长。
  • SPJ一定要谨慎出啊，好多问题，背大锅了。接下来会写一下每道题目的Idea和标程长度，还有一些出题想法。

每道题目

A 糖糖糖果

标程字节：805，难度定位：Medium-

很显然的动态规划做法，其中k最多只能是$\frac n2$，那么动态规划的状态就可以很简单的表示出来。为了让大家不开long long，我已经将$a_i$很明确的缩小到$\sum a_i \leq 1e9$的范围，但还是有人被卡了空间。
$$
dp[i][j][0]表示第i天的第j场交易买入,dp[i][j][1]表示第i天的第j场交易卖出 \
dp[i][j][0]=max(dp[i-1][j][0], dp[i-1][j-1][1]+a[i]-p) \
dp[i][j][1]=max(dp[i-1][j][1],dp[i-1][j][0] -a[i])
$$

这道题是最早出的一道题，想法来源来自leetcode系列题目买卖股票的最佳时间。

B 宝宝宝可梦

标程字节：1124，难度定位：Medium

很显然的是，默认所有的边都能选到，然后就开始割边啦。

这道题是最晚出的一道题，因为整套题目出完发现少了一道图论题。顺便自己也犯懒了，就决定出一道无向图的全局最小割的模板题。本来定位也是一道Medium题，以为大家能huahuahua乱做，没想到都没几个人尝试这道题，最后没人通过。但模板题感觉除了冷门模板之外其他模板出了就太显然了，很难拿捏这个难度定位。

顺便说一下，这道题是可以并查集贪心贪过去的，网上的几道无向图最小割的模板都能用这个方法贪过去。无敌的验题人！

C 铁铁贴贴

标程字节：630，难度定位：Easy

一道签到题，时间和范围的限制都给了一些提示，利用抽屉原理，对所有数字都依次对$c$取模，如果哪个抽屉大于等于两个，那么就输出该抽屉里的所有数字即可。

该死，为什么签到题有这么多问题。HustOJ的SPJ怎么这么难，顺便加上HustOJ的SPJ代码。并且数据还出弱了，没有考虑到抽屉数量大于$c+1$的情况，虽然不影响签到就是了。（为什么很多人输出字符串不喜欢输出个回车啊= =、）

D 数字大战

标程字节：4170，难度定位：Medium+

一道显然的码农题，出出来就是为了让大家慢慢写代码的。由于评测机太拉了，就有点卡常，本来实际上是没有卡常的。时间限制也上调到了15s，就是为了让大家通过，显然，大家都不怎么会线段树，但是有的人的快读也被评测机卡了，非常对不起。

做法很简单，观察到每次虽然都是乘法操作和立方操作，但是都只会对对应素数的指数进行加法或乘法操作，所以我们为每个素数建立一个线段树，每次维护的就是这个素数的指数。最后统计答案的时候再统一回复一下即可。

出题想法是打CCPC威海的时候，VCode笑了一下说不能有一个数据结构可以维护全体加1和全体乘积的吧，然后我就很快想到了这道题，非常的简单啊，果然卡了不少人。

E 金牌厨师

标程字节：504，难度定位：hard

一道状压DP题，需要一个长度为1 << 26的dp数组。那么就有状态转移为
$$
pos = 1 << (s[0] - ‘a’) \
\forall i \in [1, n) \
pos = 1 << (s[i] - ‘a’) \
dp[pos] = min(dp[pos], dp[pos & (1 << j)] + 1) \
其中j为枚举的状态
$$

PrimoPan丢给我想的一道题，稍微改改就成了这道防穿题，状压的思想也太妙了，可惜没人做。非常伤心。

G 智慧树

标程字节：1224，难度定位：Medium+

一道斐波那契数列的变种，加上了前偶数项的和也为偶数的要求，不难发现是n*6求和。想着太简单了就套了个欧拉降幂，纯模板题，没什么说法。

又是一道码农题，很没有说法，但是大家都不来做，那能咋办呢。

B Special Numbers

题意：

定义一种特殊数字序列，仅能通过n的不重复次幂之和求得，求第k个数字是什么。比如n=3的时候，该特殊数列为$[1=3^0,3=3^1,4=3^0+3^1,9=3^2,…]$

题解：

观察可以知道，这个特殊数字序列实际上就是代表的二进制数字，将幂数存下来发现是$[0,1,10,11,100,…]$那么就很简单了，对k进行二进制拆分即可，然后按n幂次还原。

C Make Them Equal

题意：

长度为n的字符串s，算出将其全部替换成指定字符的操作数。定义一次操作为：选择一个数字x，对于每一个位置i如果i % x != 0，那么就可以替换。

题解：

又是一道人类智慧题，打个表就可以发现，用n和n - 1可以完成所有操作。如果原串本身就都是指定字符那么就需要0次，如果存在某个位置x为指定字符且$\frac{n}{x}=1$，那么就只需要1次。

D The Number of Imposters

题意：

有n个人，身份不确定有好人有坏人，好人只说真话，坏人只说假话。有m条信息指认消息，形如a说b是好人，或者a说b是坏人。需要求出最多的可能是坏人的数量。如果矛盾输出-1。

题解：

一眼并查集，对于每个人x，可以拆成xc=x和xi=x+n两个身份。如果a说b是好人，那么就看ac和bi是不是处于一个集合，之后将ac,bc和ai,bi连在一起；如果a说b是坏人，那么就看ac和bc是不是处于一个集合，之后将ac,bi和ai,bc连在一起即可。链接过程算size。
