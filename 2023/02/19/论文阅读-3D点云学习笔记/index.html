3D点云学习笔记

3D点云

工具

  • PointCompare：3D点云可视化展示工具，并且提供了降采样的窗口
  • 本地标注工具：需要简单在本地配置环境的点云标注工具，注意输入点云不宜过大，需要进行降采样之后再由其标注
  • S3DIS：斯坦福数据集，需要注意的是斯坦福数据集本身有小错误（Stanford3dDataset_v1.2_Aligned_Version\Area_5\hallway_6\Annotations\ceiling_1.txt中的第180389行，需要手动删除）
  • pcl_pcd：胡师兄写的小工具，用来将二进制的pcd格式的点云转换成int的pcd格式
  • PCD_Data_Changer：我自己写的小工具，用来将PCD格式的点云转换成S3DIS格式

BaseLine

PointNet

PointNet主要是基于点云的置换不变性和旋转不变性提出的网络，也是后续所有点云网络的basebone。

网络变化相对简单，如果Batch_size为n，那么就是对原本的$n\times3\times2500$的数据做一次卷积，卷积到$n\times64\times2500$，然后卷积到$n\times128\times2500$，之后卷积到$n\times1024\times2500$。做完卷积后，进行一次最大池化，将数据压缩至$n\times1024\times1$，用来满足置换不变性。卷积使用的一层卷积。

俩个T-Net是用来旋转校正位置，满足旋转不变性。

分类问题直接过一个mlp分类即可，对于分割问题，将$n\times64$和$n\times1024\times1$进行拼接，再通过mlp来完成分割。

PointNet++

PointNet无法获得局部特征，很难应用于复杂场景。PointNet++提出了两种方法，第一种是利用空间距离，第二种是利用一种自适应密度的特征提取方法。

PointNet++需要解决两个关键的问题：第一，如何将点集划分为不同的区域；第二，如何利用特征提取器获取不同区域的局部特征。

PointNet++使用PointNet来作为特征提取器。在PointNet的基础上加入了多层次结构，使得网络在越来越大的区域上提供更高级别的特征，即CNN的多层感受野思想。

Sample Layer：使用最远点采样方法选择出$N$个点；Grouping Layer：对于每个被选择的点，将其在某个半径的球内找出最多$K$个点作为一组；PointNet Layer：提取每组的特征。

分类网络：逐层提取局部特征，最后总结出全局特征输出分类结果；分割网络：提取全局特征，先做反向插值，（还有一个skip操作将一步的特征直接拼接），再上采样。

对于非均匀的点云，有两种方法。一个是MSG：对于同一个中心点，如果使用3个不同尺度的话，就分别找围绕每个中心点画3个区域，每个区域的半径及里面的点的个数不同。对于同一个中心点来说，不同尺度的区域送入不同的PointNet进行特征提取，之后拼接，作为这个中心点的特征；一个是MGR：对不同尺度的组进行拼接，对于低的先做pointnet提取再和高尺度的一起提取，似乎类似ResNet的跳连接。

还提出了random input dropout（DP）的方法，就是在输入到点云之前，对点集进行随机的Dropout，比例使用了95%，也就是说进行95%的重新采样。

大规模点云语义分割

RandLA-Net

PointNet，PointNet++等方法处理大规模点云时，往往都是将点云分割成小点云块，再从中采样点输入网络，这不太能够有效学习到空间中的网络结构。文章做了对比试验，通过放大分割点云的大小，但是并不能解决这个问题。

一种处理方法是SPG使用超点超图的方式来表征大场景点云。后续会进行一下学习。

本文的主要贡献：对现有的降采样方法进行了分析和比较，认为随机降采样是一种适合大规模点云高效学习的方法；提出一种有效的局部特征聚合模块，通过逐步增加每个点的感受野来更好地学习和保留大场景点云中复杂的几何结构；RandLA-Net在多个大场景点云的数据集上都展现出了非常好的效果以及非常优异的内存效率以及计算效率。

Random Sampling (RS)：随机降采样均匀地从输入的 N 个点中选择 K 个点，每个点具有相同的被选中的概率。RS的计算复杂度为常数级别复杂度；其计算量与输入点云的总点数无关，只与降采样后的点数K有关，也即常数时间复杂度。因而具有非常高的效率以及良好的可扩展性。

随机采样会可能会失去有用的信息，所以本文提出了一种局部特征提取器。包括三个模块，局部空间编码器、Attentive Pooling和扩张残差块。

局部空间编码（LocSE）：此模块用于显式地对输入的点云的三维坐标信息进行编码。不同于直接将各个点的三维坐标作为一个普通的通道特征输入到网络中，LocSE模块旨在显式地去编码三维点云的空间几何形状信息，从而使得网络能够从各个点的相对位置以及距离信息中更好地学习到空间的几何结构。具体步骤为：用$KNN$为每一个点$p_i$找到欧氏距离最近的$K$个点；对于$p_i$的$K$个最近的$K$个点，将中心点的三维坐标$p_i$和邻域点的三维坐标$p_i^K$相对坐标以及欧氏距离连接到一起，$r_i^K=MLP(p_i\oplus p_i^K\oplus(p_i-p_i^K)\oplus ||p_i-p_i^K||)$，将邻域点对应的点特征和相对位置拼接到一起，就可以得到最后的点特征。

Attentive pooling：该模块用来提取特征，主流的启发式方法为Max / Mean / sum之类的。

**扩张残差块(Dilated Residual Block)**：扩张残差块最终使用两组LocSE和attentive pooling单元，以平衡最终的分割性能以及计算效率。首先做一次KNN特征提取，再做一次特征提取，将K进化到K^2。

将随机采样以及局部特征聚合模块组合到一起，基于标准的encoder-decoder结构组建了RandLA-Net。网络的详细结构如下图所示，可以看到，输入的点云在RandLA-Net中持续地进行降采样以节约计算资源及内存开销。此外，RandLA-Net中的所有模块都由简单高效的feed-forward MLP组成，因此具有非常高的计算效率。最后，在解码器中的上采样阶段，不同于广泛采用的三线性插值(trilinear interpolation)，我们选择了更加高效的最近邻插值(nearest interpolation)，进一步提升了算法的效率。

  • 对于三维点云语义分割任务而言，与其在被切割的点云上提出非常复杂的算法来提升性能，不如直接尝试在大场景点云上进行处理，这样更加有实际意义。
  • 三维点云分割网络的scalability也是实际应用中一个比较重要的点。i.e., 理想情况下train好的网络应该可以用于inference任意点数的输入点云，因为每个时刻采集到的点云的点数不一定是相同的。这也是RandLA-Net没有使用全局特征的原因，i.e. 确保学到的参数是agnostic to number of points.

SQN

SQN由两个部分组成，第一个部分是点局部特征提取器，第二个部分是点特征查询网络。

点局部特征提取器用来提取所有点的局部特征。（这个部分是可以自由替换的，本文采用RandLA-Net作为特征提取器），该提取器由四层伴有随机采样（RS）的局部特征聚合（LFA）组成。这些网络通常将多个编码层叠加在一起并通过降采样操作提取分层局部特征。

点特征查询网络，该查询网络旨在收集尽可能多的相关特征，并使用可用的稀疏信号进行训练。以特定的 3D 查询点为输入，获取与该点相关的一组学习到的点特征。该网络由空间邻近点特征搜索（Searching Spatial Neighbouring Point Features）、插值查询点特性（Interpolating Query Point Features）和推断查询点语义（Inferring Query Point Semantics）这三部分构成。

  • 空间临近点特征搜索：根据点之间的欧式距离再特征提取器的4个编码器层中搜索距离查询点最近的k个点，k为超参数。
  • 插值查询点特征：对于每一层的特征，查询的 K 个向量被压缩成查询点 p 的一个紧凑表示（compact representation）。为了简单起见，论文应用三线性插值方法，根据 p 和 K 个点之间的欧氏距离计算 p 的特征向量。最后，将四个层次特征向量拼接在一起，表示整个三维点云中所有相关的点特征。
  • 推断查询点语义：在得到查询点 p 唯一且具有代表性的特征向量后，我们将其输入到一系列 MLP 中，直接推断出点的语义类别。
