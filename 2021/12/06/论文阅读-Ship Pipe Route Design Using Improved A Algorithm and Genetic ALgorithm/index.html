Ship Pipe Route Design Using Improved A* Algorithm and Genetic Algorithm

需要知道的一些东西

  • SPRD：船舶管道路径设计问题；确定性路由算法，时间效率更好；非确定性路由算法，优化能力更好；混合路由算法，做一个balance；

路由空间表示

  • 路由空间用长方体表示，根据待布线管道的管径值和管道间的最小间距将空间分解为网格。将已经被覆盖或者不可以放置管道的地方标记一下，其他网格就是自由网络。用$S_{grids}$表示空间分解生成的网格集合，$S_{obs}$表示障碍物空间的长方体集合。

  • 判断障碍物网格或自由网格的过程可以简化为：遍历每个网格，对于选择的网格，判断是否有顶点落在$S_{obs}$内，比较坐标。如果存在，就是障碍网格，否则为自由网格。遍历所有网格之后，设置如果靠近墙壁地板等地为低能量，锅炉周围电器设备周围为高能量值，通过能量值动态调整管道通过的可能性。

  • 在布置不同管径值的管道时，根据与网格边长相等的最小管径将布局空间分解成网格。当铺设直径较大的管道时，空格内的障碍物应向外延伸一定距离。可以形成如下公式：$N_p=\left{ \begin{array}{lr} int(\frac{D_p}{2L}),\frac{D_p}{2L}-int(\frac{D_p}{2L})\leq 0.5 \ int(\frac{D_p}{2L})+1,\frac{D_p}{2L}-int(\frac{D_p}{2L}) > 0.5\end{array} \right.$，其中，$N_p$表示需要被障碍物扩展的网格数量，$D_p$表示管道最大直径，$L$表示网格边长。直径包括管道之间的最小允许距离。

    障碍物扩展过程：预处理过程中，将所有处于障碍物内部或边界处的网格标记为障碍物网格。基于此，扩展过程中只需要遍历$S_{grids}$如果是障碍物就跳过，否则按照公式检查是否需要扩展，如果需要则标记为临时障碍网格，扩展中不允许通过，否则允许通过。初始喷嘴位置可能会被障碍物延伸覆盖，有必要将每个管道喷嘴的初始位置沿原方向向外延伸，尹成心得连接点，否则，每次选管道前都将其放入障碍物集合中。

管道表示

  • 管道路径可以用每个网格的中心点坐标序列表示，定义为$p_1(x_1,y_1,z_1)\rightarrow p_2(x_2,y_2,z_2)\rightarrow … \rightarrow p_n(x_n,y_n,z_n)$。$p_1$是起始点，$p_n$是目标点，在网格分解空间中，可以将网格坐标序列转换为网格位置序列$p_1(r_1,c_1,l_1)\rightarrow p_2(r_2,c_2,l_2)\rightarrow … \rightarrow p_n(r_n,c_n,l_n),|r_i-r_{i-1}|+|c_i-r_{i-1}|+|l_i-l_{i-1}|=1,(1<i\leq n)$，根据管道的半径向外扩展路径可以得到最终形状。如果为了指明某个管道，在每个字母上加一个$p_1^i$即可。
  • 如果俩管道的某个节点在网格里的曼哈顿距离为1，那么就是相邻的，需要重新安排

约束和目标

  • 连接管口，避免任何障碍物干扰；管道与刚性结构正交布置；满足两个相邻弯头之间的最小长度要求；尽可能最小化路径长度；尽可能减少玩去次数；将管道沿墙壁，地板或设备铺设，取得更好地支撑；尽可能平行排列；

数学建模

  • 约束和目标分别用A*算法和遗传算法解决。
  • 目标函数为$Min.f_{Objective}(path)=\alpha \times f_L(path) + \beta \times f_B(path)+\gamma \times f_E(path) - \delta \times f_N(path)$，其中$f_L(path)$为路径上网格数；$f_B(path)$为路径上弯曲数量；$f_E(path)$为路径上能量值之和；$f_N(path)$衡量当前管道和路由管道的平行路由效果。该值越大，相邻的网格对越多，其中各希腊字母是用户自行选择的，用来平衡。

改进A*算法

  • $g(n)=a\times L_{cost}(n) + b\times E_{cost}(n)+ c\times B_{cost}(n)$，其中$a,b,c$是关心权重因子，$L_{cost}(n),E_{cost}(n),B_{cost}(n)$分别是长度成本，能量成本和弯曲数成本。每次在搜索过程中都需要对应的更新

  • $h(n)=abs(x_n-x_t)+abs(y_n-y_t)+abs(z_n-z_t)$，采用曼哈顿距离，$(x_n,y_n,z_n)$是网格位置，$(x_t,y_t,z_t)$是目标网格。可以适当放大h函数。

  • $f(n)=g(n)+w\times h(n)=a\times L_{cost}(n)+b\times E_{cost}(n)+c\times B_{cost}(n)+w\times(abs(x_n-x_t)+abs(y_n-y_t)+abs(z_n-z_t)), w \geq 1$，如果b和c为0，则搜索只受路径长度的引导；当b和c为正值时，考虑弯曲次数和沿支架走线的约束。

  • 开三个三维数组，close_map，open_map和dir_map。close_map表示是否被搜过，open_map用来改进效率，dir_map用来保持路径上网格的顺序。管道的网格路径可以使用dir_map和管道喷嘴来跟踪。

  • $$
    输入:起始网络s,目标网格e,S_p[M][N][L]表示路由空间,M、N、L分别表示空间的层号、列号和行号\
    输出:管道路径\
    步骤:(1)创建一个空的优先队列,openmap[c.layer][c.column][c.row] \leftarrow 0,\closemap[c.layer][c.column][c.row] \leftarrow 0,\dirmap[c.layer][c.column][c.row] \leftarrow 1\
    (2)计算初始结点的f(s),将(f(s),s)放入pq,计算s的实际消耗 openmap[s.layer][s.column][s.row]\leftarrow g(s) \
    (3)如果pq非空的进入(4)到(17)循环\
    (4)取pq顶部元素v \
    (5)如果v是目标网格,停止搜索并通过dirmap建立管道路径,如果v不是继续下面步骤 \
    (6)设置状态为已经搜索过 openmap[v.layer][v.column][v.row] \leftarrow 0, clsoemap[v.layer][v.column][v.row] \leftarrow1 \
    (7)令v_0 = v \
    (8)探索v周围六个正交方向的网格,继续下面9到17步骤 \
    (9)如果满足以下abc三种情况,继续步骤10到16,否则继续步骤17 \
    (a)(v.layer + k, v.column + j, v.row +i)是一个可以使用的有效坐标 \
    (b)S_p[v.layer+k,v.column+j,v.row+i]是一个自由网格,不在任何障碍物或者其他路由管道之中 \
    (c)closemap[v.layer+k,v.column+j,v.row+i]=0 \
    (10)假设v是一个新位置,v(layer,column,row)\leftarrow v(layer+k,column+j,row+i) \
    (11)判断是否形成一个新的弯板,如果形成了计算到上个弯板的距离d,如果未形成弯曲或者距离d\geq L_{min}执行12否则17\
    (12)尝试计算g(v),并给dist \
    (13)如果openmap[v.layer][v.column][v.row]是0继续14否则15 \
    (14)计算g(v),h(v)并更新f(v).并推到pq里 \
    (15)如果openmap对应的值大于等于dist,那么就是新的路径更好 \
    (16)在pq里找到v更新对应的f值 \
    (17)更新转8 \
    (18)未找到,返回故障信息
