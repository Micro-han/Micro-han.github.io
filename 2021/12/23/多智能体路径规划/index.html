多智能体强化学习路径规划

概要

  • 参考了三篇知乎文章，主要学习一下这个思想建模。

A*+RL1

Action

用强化学习来决定执行传统路径规划算法的动作还是其他动作。传统路径规划算法比如A*算法，其他动作比如前进后退左拐右拐停止等等。

标识 动作
$a_0$ 采用传统路径规划算法
$a_1$ 停留
$a_2$ 朝前移动
$a_3$ 左转
$a_4$ 右转
$a_5$ 向后移动

其中，$a_0$就是A*算法的动作，强化学习就是学习$a_0\leftrightarrow a_5$这六个离散动作。

State

如果某个方向上存在障碍，感知到的东西就是$(0,d_i)$，如果是其他智能体，感知到的东西就是$(1,d_i)$，如果啥都没有就是$(0,0)$。$d_i$为该智能体到该方向上障碍的距离。

Reward
$$
r_t=r_{navigation}+r_{scenario}+r_{penalty}
$$
第一项，如果是A*算法，那么就是$a_0$，附加一个正向奖励。$r_{navigation}=\left{ \begin{array}{1}r_0,if & a_0\ 0, else \end{array} \right.$

第二项，如果才用的动作撞到了障碍物或者墙壁或者其他智能体，就扣一个大分数，到了终点就加一个大分数。$r_{scenario}=\left{ \begin{array}{1} r_{negative},if & collides \ r_{positive}, if & arrive \ 0, else \end{array} \right.$

第三项，是每一项的惩罚系数，小于0。

A*+RL2

State

Channel1，障碍物和其他车的位置；Channel2其他车的轨迹序列，同一辆车的轨迹序列的值，不同时刻不同；A*规划的路径，但不是时时刻刻规划，频率很低。

Action

九个动作，八个方向加上停。

Reward

$R=r_s+r_c+r_o+r_g+\lambda r_f$，$r_s\in {-0.1, -0.5}$为每一步的惩罚系数，$r_c=-5$碰撞惩罚，$r_o=-0.3$震荡惩罚，$r_f\in -min_{p\in S} ||P_a-P||_2$，偏离A*规划轨迹的惩罚，$r_g=30$到达终点奖励。

多智能体强化学习路径规划

全局静态路径规划方法

  • 全局静态路径规划分为位置空间表示和路径搜索算法两部分。位置空间是与实际环境对应的、能够被计算及处理的数据结构；路径搜索算法则是应用于该结构的算法。

  • 传统位置空间将实际环境划分为可行区域和障碍物区域，并将可行区域看作各向同性，区域内的安全性、通行性一致。这会导致生成的路径距障碍物过近。

  • 针对此缺陷，快速行进平方法将可行区域看作各向异性的，认为某点的安全性和该点距障碍物的距离线性相关，倾向于让智能体沿着最安全的区域行驶。但距障碍物越远，路径长度会相应增加。

  • 提出可变快速行进平方法认为某点的安全性和该点距障碍物的距离正相关，通过比例缩放、阈值处理、函数变换这三种方式改变可行区域的各向异性，使最终生成的路径能够在安全性和长度之间进行权衡。

局部动态路径规划方法

  • 针对此问题，提出有限快速行进法。对智能体编队中某个成员进行局部动态路径规划时，将其它成员也看作动态障碍物，在每个动态障碍物的避碰领域内应用有限快速行进法，与全局静态路径规划中的安全地图叠加，用于生成优化路径。

  • 全局路径规划

    快速行进法构建的势场具有全局最小值的特点，唯一的极小值点就是起始点，避免了局部最小值，保证了算法的完整性。对快速行进法建立的势场应用梯度下降法（Gradient Descent method），通过跟随梯度的下降方向，即能找到最短的无障碍路径。

    用于最终求解最优路径的梯度下降法，函数的梯度方向是函数值增加的最快方向，与之相反，负梯度方向是函数值减小的最快方向。于是可以将负梯度方向作为一维搜索的方向，用于解决优化问题，这种方法因此被称作梯度下降法。

    快速行进法将可行区域看作各向同性，区域内的安全性、通行性一致。这使得生成的路径距静态障碍物过近。针对此缺陷，快速行进平方法将可行区域看作各向异性的，认为某点的安全性和该点距障碍物的距离线性相关，倾向于让智能体沿着最安全的区域行驶。但距障碍物越远，路径长度会相应增加。提出的可变快速行进平方法认为某点的安全性和该点距障碍物的距离正相关，通过比例缩放、阈值处理、函数变换这 3 种方式改变可行区域的各向异性，即位姿空间对应的速度图，使最终生成的路径能够在安全性和路径长度之间进行权衡。

  • 局部路径规划

    提出用于构建动态目标的有限快速行进法（Finite Fast Marching method）。有限快速行进法中界面只在一定区域内进行传播。因为界面需要到达的区域面积大大减少，有限快速行进法的计算量也相应减少，而算法原理及计算步骤并没有改变，这意味着有限快速行进法的运算速度得到提升。这增强了算法处理动态避碰问题的能力，因为处理动态障碍物需要算法具有较高的运算速度。

    从快速行进法的原理来看，界面传播的时间只与距离有关，具体指的是源点和界面最终到达点之间的距离。快速行进法的计算时间和 R 成正比，有限快速行进法的计算时间和r 成正比，两者之间的比值为 R > r。

  • 路径规划过程

    采用的为领导者-跟随者模型，使用的路径规划算法为实时在线运行，以尽可能保持编队结构的稳定。领导者的目标点是整个任务的目标点，并且是固定的，除非在行驶过程中任务发生变化。与领导者不同，跟随者的目标点会根据编队结构在每个控制周期重新规划。基于领导者和跟随者各自的目标点，快速行进法循环应用于每个编队成员，以实时规划出相应的无碰路径。

    在每个控制周期 t，均先对领导者智能体进行路径规划。算法首先根据任务需要及实际环境对位姿空间应用可变快速行进平方法，得到静态环境地图，并将其缩放至 0-1 范围内。因为静态环境在整个路径规划过程中不发生变化，故可以将其记为 Mstatic 并存储起来，一次计算，多次使用。然后，根据动态障碍物、编队中跟随者智能体的瞬时位置及速度，使用有限快速行进法对动态目标进行建模，得到动态环境地图，并将其缩放至 0-1 范围内。叠加静态环境地图和动态环境地图后，得到最终的环境地图，对其应用快速行进法和梯度下降法，即能得到领导者智能体的最优路径。

    一旦确定了领导者智能体的最优路径，算法则循环为跟随者智能体计算最优路径。这一过程与计算领导者智能体最优路径的类似，但因为跟随者智能体的目标点在每个控制周期 t 都需要重新计算，可能出现目标点落入障碍物区域的情况，导致路径规划失败，因此，采用线形比例缩放的方法将子目标点调整到新的合适位置，尽量降低对智能体编队整体表现的影响，这样处理的好处是计算量小，算法实时性高。

    子目标点重新规划算法中的缩减因子 Rd Scalar 需根据无人艇的动力学特性而定，如果智能体具有较强的机动性，则可以将Rd Scalar设定为如 0.1 这样的较小值，以充分利用智能体的机动性将距离最大限度的缩短。根据每个编队成员子目标点，编队路径规划为编队中每个跟随者智能体计算最优轨迹。当所有跟随者智能体的轨迹均更新完成后，即标志着当前规划周期的结束。得到的轨迹会被传递至路径执行层以控制编队成员的行驶。这整个过程会持续进行直到领导者智能体达到其最终的目标点。
